# Azure AI Foundry Agents and Foundry IQ

Research findings based on Microsoft's official documentation.

## **Azure AI Foundry Agents - Overview**

Azure AI Foundry Agents is a comprehensive service for building, deploying, and managing AI agents at enterprise scale. It's part of the Microsoft Foundry platform.

### **Key Concepts**

**Types of Agents:**
1. **Prompt-based agents** - Declaratively defined single agents with model configuration, instructions, tools, and natural language prompts
2. **Workflow agents** - Advanced agentic workflows with sequences of actions or multiple orchestrated agents
3. **Hosted agents** - Containerized agents developed in code via supported frameworks, deployed and managed by Foundry Agent Service

### **Agent Factory Architecture**

The service follows a factory-line approach with 6 key stages:

1. **Models** - Choose from GPT-4o, GPT-4, GPT-3.5, Llama, and others
2. **Customizability** - Fine-tune with distillation or domain-specific prompts
3. **Knowledge and Tools** - Access to Bing, SharePoint, Azure AI Search, Logic Apps, Azure Functions, OpenAPI
4. **Orchestration** - Connected agents handle tool calls, state management, retries, and logging
5. **Observability** - Full conversation-level visibility with Application Insights integration
6. **Trust** - Enterprise security with Entra ID, RBAC, content filters, encryption, network isolation

### **Development Options**

- **Portal-based**: Create agents in the Foundry portal with visual playground
- **Code-first**: Python and C# SDKs with REST API support
- **Agent Framework**: Integration with Microsoft Agent Framework for persistent agents

## **Foundry IQ Knowledge Base - Overview**

Foundry IQ is powered by **Azure AI Search** and provides intelligent knowledge retrieval capabilities for agents through agentic retrieval pipelines.

### **What is Foundry IQ?**

It's a knowledge management system that enables agents to:
- Ground responses in enterprise data
- Provide factual, citation-backed answers
- Search across multiple data sources
- Use advanced retrieval techniques

### **Core Components**

**1. Knowledge Sources** (What to retrieve)
- Azure Blob Storage
- SharePoint (including remote connections)
- Azure AI Search indexes
- Microsoft OneLake
- Other supported sources

**2. Knowledge Bases** (How to retrieve)
- Orchestrates query planning and decomposition
- Executes subqueries using keyword, vector, or hybrid search
- Applies semantic reranking
- Synthesizes unified responses with source references

### **How It Works - Agentic Retrieval Pipeline**

When a user submits a complex query:
1. **Query Planning** - LLM decomposes user query into subqueries
2. **Parallel Processing** - Subqueries execute simultaneously using keyword/vector/hybrid techniques
3. **Semantic Reranking** - Identifies most relevant results
4. **Answer Synthesis** - Combines results into unified response with citations

### **Output Modes**

- **Extractive Data** - Verbatim content from sources (recommended for agent integration)
- **Answer Synthesis** - Natural language responses generated by LLM

## **Building an Agent with Foundry IQ Knowledge Base**

### **Prerequisites**

1. **Azure AI Search service** with:
   - Knowledge base created
   - One or more knowledge sources configured
   - Semantic ranker enabled
   
2. **Microsoft Foundry project** with:
   - LLM deployment (e.g., gpt-4.1-mini)
   - Proper role assignments

3. **Authentication & Permissions**:
   - **Azure AI User** role on Foundry project
   - **Azure AI Project Manager** role to create connections
   - **Search Index Data Reader** role for search service managed identity

### **Integration Architecture**

The integration uses **Model Context Protocol (MCP)** to connect agents with knowledge bases:

```
Agent → MCP Tool → Knowledge Base → Knowledge Sources → Data
```

### **Key Implementation Steps**

**Step 1: Create a Knowledge Base in Azure AI Search**
- Define knowledge sources (blob, SharePoint, search indexes)
- Configure vectorizer for embeddings
- Set up semantic ranking
- Choose output mode (extractive or answer synthesis)

**Step 2: Create Project Connection**
- Create `RemoteTool` connection in Foundry project
- Target MCP endpoint: `{search_endpoint}/knowledgebases/{kb_name}/mcp?api-version=2025-11-01-preview`
- Use project managed identity for authentication

**Step 3: Create Agent with MCP Tool**
- Define agent with system instructions
- Add MCP tool pointing to knowledge base
- Configure `knowledge_base_retrieve` tool (the only supported MCP tool currently)
- Set `require_approval` parameter (never, always, or per-tool list)

**Step 4: Optimize Agent Instructions**

Recommended template:
```
You are a helpful assistant that must use the knowledge base to answer all questions from user. 
You must never answer from your own knowledge under any circumstances.
Every answer must always provide annotations using the MCP knowledge base tool and render 
them as: 【message_idx:search_idx†source_name】
If you cannot find the answer in the provided knowledge base you must respond with "I don't know".
```

**Step 5: Invoke Agent**
- Create conversation session
- Send user query
- Agent orchestrates MCP tool calls to knowledge base
- Returns synthesized answer with citations

### **Advanced Features**

**SharePoint Knowledge Sources:**
- Requires additional `x-ms-query-source-authorization` header
- Supports per-user trimming with ACL fields

**Security:**
- Role-based access control throughout
- Managed identity authentication
- Token-based security
- No credential management in code

**Deployment:**
- One-click deployment with Azure Developer CLI (azd)
- Container Apps hosting for hosted agents
- Automatic infrastructure provisioning with Bicep templates

## **Key Benefits of This Architecture**

1. **Separation of Concerns** - Knowledge base handles retrieval logic, agent handles conversation and synthesis
2. **Reusability** - Knowledge sources and bases can be shared across multiple agents
3. **Scalability** - Cloud-native architecture with managed services
4. **Transparency** - Citation-backed responses with source attribution
5. **Enterprise-Grade** - Built-in security, monitoring, and compliance features
6. **Version Control** - Agent versioning with immutable versions and draft states

## **Python Code Example - Creating Agent with Foundry IQ**

### **Step 1: Create Project Connection**

```python
import requests
from azure.identity import DefaultAzureCredential, get_bearer_token_provider

# Provide connection details
credential = DefaultAzureCredential()
project_resource_id = "{project_resource_id}"  # e.g. /subscriptions/{subscription}/resourceGroups/{resource_group}/providers/Microsoft.MachineLearningServices/workspaces/{account_name}/projects/{project_name}
project_connection_name = "{project_connection_name}"
mcp_endpoint = "{search_service_endpoint}/knowledgebases/{knowledge_base_name}/mcp?api-version=2025-11-01-preview"

# Get bearer token for authentication
bearer_token_provider = get_bearer_token_provider(credential, "https://management.azure.com/.default")
headers = {
    "Authorization": f"Bearer {bearer_token_provider()}",
}

# Create project connection
response = requests.put(
    f"https://management.azure.com{project_resource_id}/connections/{project_connection_name}?api-version=2025-10-01-preview",
    headers=headers,
    json={
        "name": "project_connection_name",
        "type": "Microsoft.MachineLearningServices/workspaces/connections",
        "properties": {
            "authType": "ProjectManagedIdentity",
            "category": "RemoteTool",
            "target": mcp_endpoint,
            "isSharedToAll": True,
            "audience": "https://search.azure.com/",
            "metadata": {"ApiType": "Azure"}
        }
    }
)

response.raise_for_status()
print(f"Connection '{project_connection_name}' created or updated successfully.")
```

### **Step 2: Create Agent with MCP Tool**

```python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import PromptAgentDefinition, MCPTool
from azure.identity import DefaultAzureCredential

# Provide agent configuration details
credential = DefaultAzureCredential()
mcp_endpoint = "{search_service_endpoint}/knowledgebases/{knowledge_base_name}/mcp?api-version=2025-11-01-preview"
project_endpoint = "{project_endpoint}"  # e.g. https://your-foundry-resource.services.ai.azure.com/api/projects/your-foundry-project
project_connection_name = "{project_connection_name}"
agent_name = "{agent_name}"
agent_model = "{deployed_LLM}"  # e.g. gpt-4.1-mini

# Create project client
project_client = AIProjectClient(endpoint=project_endpoint, credential=credential)

# Define agent instructions
instructions = """
You are a helpful assistant that must use the knowledge base to answer all the questions from user. You must never answer from your own knowledge under any circumstances.
Every answer must always provide annotations for using the MCP knowledge base tool and render them as: `【message_idx:search_idx†source_name】`
If you cannot find the answer in the provided knowledge base you must respond with "I don't know".
"""

# Create MCP tool with knowledge base connection
mcp_kb_tool = MCPTool(
    server_label="knowledge-base",
    server_url=mcp_endpoint,
    require_approval="never",
    allowed_tools=["knowledge_base_retrieve"],
    project_connection_id=project_connection_name
)

# Create agent with MCP tool
agent = project_client.agents.create_version(
    agent_name=agent_name,
    definition=PromptAgentDefinition(
        model=agent_model,
        instructions=instructions,
        tools=[mcp_kb_tool]
    )
)

print(f"Agent '{agent_name}' created or updated successfully.")
```

### **Step 3: Invoke Agent with Query**

```python
# Get the OpenAI client for responses and conversations
openai_client = project_client.get_openai_client()

# Create conversation
conversation = openai_client.conversations.create()

# Send request to trigger the MCP tool
response = openai_client.responses.create(
    conversation=conversation.id,
    input="""
        Why do suburban belts display larger December brightening than urban cores 
        even though absolute light levels are higher downtown?
    """,
    extra_body={"agent": {"name": agent.name, "type": "agent_reference"}},
)

print(f"Response: {response.output_text}")
```
